## 🔴 CRITICAL — fix NGAY (8 issues, đa phần security/data integrity)

### 1. Auth bypass — `POST /notification/dev/seed-notifications` không check secret
- **File**: `backend/src/notification/notification.controller.ts:641-720`
- **Exploit**: ai cũng có thể `curl` endpoint này → spam 7 push notification + FCM cho user bất kỳ. Cũng leak userId/name/post title của bất kỳ user nào → enumeration oracle.
- **Fix**: thêm `if (!this.checkDevSecret(body.secret)) return { error: 'unauthorized' };` (giống 11 endpoint dev khác trong cùng file).

### 2. PII leak — `GET /user/:id` không auth, trả email + phone
- **File**: `backend/src/user/user.controller.ts:37-40` + `user.service.ts:138-164`
- **Exploit**: `curl https://api.traotay.com.vn/user/<id>` → email + phone. Combine với `GET /user` (mục #3 dưới) → harvest toàn bộ database email/phone trong vài phút. Vi phạm Nghị định 13/2023 về bảo vệ dữ liệu cá nhân.
- **Fix**: chỉ trả email/phone khi `viewerId === id` (owner) hoặc admin.

### 3. PII leak #2 — `GET /user` list toàn bộ user, không pagination
- **File**: `backend/src/user/user.controller.ts:60-64`
- **Exploit**: bất kỳ user đăng nhập nào (sign up bằng phone OTP là xong) get list ID của toàn bộ user → kết hợp #2 → harvest hoàn chỉnh.
- **Fix**: thêm `AdminGuard`, hoặc xoá endpoint, hoặc pagination + bỏ PII.

### 4. Privacy leak — `GET /favorite/:userId` xem favorites user khác
- **File**: `backend/src/favorite/favorite.controller.ts:21-25`
- **Exploit**: Path nhận `userId`, không verify với `req.user.id` → xem được sở thích của user khác.
- **Fix**: check `userId !== req.user.id → ForbiddenException`, hoặc đổi thành `@Get('me')`.

### 5. Categories mismatch — 7/18 danh mục Flutter bị silently rewrite "other"
- **Files**: `backend/src/post/post.service.ts:10-13` (whitelist 14 keys) vs `app/lib/data/categories.dart` (18 keys)
- **Bug**: Flutter gửi `clothing/furniture/kitchen/tools/baby/music/service` → backend không có trong whitelist → set thành `'other'` ở `post.service.ts:185`. Bài "Áo Zara" (clothing) lưu DB là `other` → search/filter không tìm được.
- **Fix**: đồng bộ key giữa Flutter và backend (recommend dùng đúng 18 key Flutter).

### 6. Province picker filter — vẫn hardcode 63 tỉnh CŨ (bug giống map_picker)
- **File**: `app/lib/widgets/province_picker_sheet.dart:77-109`
- **Bug**: Filter tìm kiếm vẫn liệt kê đầy đủ 30 tỉnh đã sáp nhập (Hải Dương, Bình Dương, Bến Tre, …). DB đã migrate sang tỉnh mới → user chọn "Hải Dương" filter → 0 kết quả.
- **Fix**: thay `_regions` cứng bằng `AppProvinces.list` (vùng miền đã có trong `home_tab.dart`).

### 7. NotificationModule 3 instance độc lập — realtime badge unread broken
- **Files**: `backend/src/chat/chat.module.ts:21`, `keyword-alert/keyword-alert.module.ts:10`, `app.module.ts`
- **Bug**: NotificationService được khai báo trùng ở 3 module → 3 instance độc lập. NotificationGateway chỉ wired vào instance trong AppModule. Khi chat/keyword-alert tạo notification → `gateway.sendToUser()` silently skip (gateway = null) → badge unread không fire realtime, user phải refresh tab Notification mới thấy.
- **Fix**: tạo `NotificationModule` export Service + Gateway → import vào ChatModule + KeywordAlertModule + AppModule.

### 8. Trust proxy missing — rate limit broken (global thay vì per-IP)
- **Files**: `backend/src/main.ts` (no `app.set('trust proxy')`), Nginx forward `X-Forwarded-For`
- **Bug**: NestJS Throttler dùng `req.ip` = `127.0.0.1` (do Nginx local) → mọi rate limit (3/min cho email OTP, 10/min cho phone, …) đang là **global cho toàn server**, không phải per-IP. Attacker hit 3 lần email-login/send → khoá toàn bộ user khỏi email login 60s. DoS auth flow giá rẻ.
- **Fix**: trong `main.ts` sau `NestFactory.create`: `app.set('trust proxy', 'loopback')`.

## 🟠 HIGH — fix trước/trong Phase 2 (10 issues)

### 9. listingType mismatch — DB có 'free'/'exchange' → crash dropdown edit
- `app/lib/screens/post/edit_post_screen.dart:29-32` chỉ có 2 option (`sell`, `give`), backend whitelist 4 (`sell`, `give`, `free`, `exchange`). Bài cũ có `listingType = 'free'` → DropdownButton throw exception khi mở Edit.
- **Fix**: hoặc bổ sung 2 option Flutter, hoặc backend bỏ `free`/`exchange` khỏi whitelist + migrate data.

### 10. Bump payment thiếu confirm dialog — tâm lý VN cần xác nhận trước trả tiền
- `app/lib/screens/post/bump_package_screen.dart:88-197`: bấm "Thanh toán 15k" → nhảy thẳng vào WebView PayOS, không có dialog "Bạn sắp trả 15.000đ — Đồng ý?"
- **Fix**: thêm AlertDialog confirm trước khi mở WebView.

### 11. Polling 6s quá ngắn cho 3G/4G VN
- `app/lib/screens/post/bump_package_screen.dart:158-186`: poll boost status 1s × 6 lần. 3G yếu thường > 10s → user thấy "Nếu bài chưa boost sau 1-2 phút..." hoang mang.
- **Fix**: tăng lên 15s + thêm tab "Lịch sử thanh toán".

### 12. ChatScreen `late _socket` crash khi token null
- `app/lib/screens/chat_screen.dart:47, 105, 165, 181`: `late IO.Socket _socket` không init nếu token null → dispose() throw LateInitializationError.
- **Fix**: đổi sang `IO.Socket? _socket;` + null check trước mọi gọi.

### 13. JWT lưu plain text trong SharedPreferences
- `app/lib/services/api_service.dart:78, 137, 168`: token JWT (sống 7 ngày) lưu SharedPreferences → Android root/iOS jailbreak/backup `/data/data` đọc được. `flutter_secure_storage` đã có trong pubspec nhưng chưa import.
- **Fix**: migrate `auth_token` sang `flutter_secure_storage`.

### 14. Dev secret hardcoded APK production
- `app/lib/services/api_service.dart:156`: `static const String _devSecret = 'traotay_dev_2024';` nằm trong APK release, dễ tìm bằng `strings`.
- **Fix**: chỉ build dev login khi `kDebugMode`, hoặc `--dart-define=DEV_SECRET=...`.

### 15. CategoryDropdown 18 mục flat — vi phạm Miller's Law
- `app/lib/screens/post/create_post_tab.dart:732-746`: 18 danh mục trong DropdownButtonFormField phẳng. Người già/sinh viên scan 18 dòng để tìm "Mẹ & Bé" → dễ chọn nhầm.
- **Fix**: thay bằng `showModalBottomSheet` grid 3 cột với icon (đã có asset).

### 16. Map picker thiếu nút "Vị trí của tôi"
- `app/lib/screens/post/map_picker_screen.dart`: mặc định mở ở Hà Nội. User TP.HCM phải zoom out → chuyển → zoom in. MapViewScreen có nút này, map_picker thì không.
- **Fix**: thêm `FloatingActionButton(Icons.my_location)` + GPS user.

### 17. Chat không upload ảnh — quan trọng cho chợ đồ cũ
- `app/lib/screens/chat_screen.dart:593-685`: input bar chỉ text. Quick reply có gợi ý "cho mình xem ảnh thực tế" (line 211!) nhưng KHÔNG có cách upload. User phải chuyển sang Zalo → mất user.
- **Fix**: thêm IconButton(camera) trong InputBar.

### 18. Splash delay 2.4s mỗi lần mở app
- `app/lib/screens/splash_screen.dart:79-82`: `await Future.delayed(2400ms)` luôn chạy. User mở app nhiều lần/ngày phải đợi mỗi lần.
- **Fix**: tách animation/navigation, navigate ngay khi auth ready.
